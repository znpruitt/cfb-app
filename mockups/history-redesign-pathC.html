What changed in this pass

Multi-line restructured rows

Championships: name + titles + years + reigning marker on line 1; career context (seasons played, win%, distinguishing fact) on line 2. Hardiman's "league's first champion" phrasing earns its own row by adding genuine historical context. Whited gets "all-time wins leader" — meaningful note that ties together two facts on the page.

Top rivalries: pair + score on line 1; games + seasons + most-recent meeting + winner on line 2. The "last met 2025 (Whited)" addition brings recency context that makes the rivalry feel alive — answering "who's hot in this rivalry right now?"

Title droughts: name + drought count on line 1; career framing on line 2 ("Never won · 3 top-3 finishes · best #2 in 2024"). Turns the section from a column of bare numbers into one that tells each chaser's story.

Movers: owner + delta on line 1; full span + rank change on line 2 with editorial annotations ("won title", etc. where applicable). The line-2 annotation hints when a mover has a championship narrative behind it.

Restored to dense table

All-time standings: brought back Seasons and Avg-finish columns from the production all-time-standings table. Now 7 columns wide, dense enough to justify the column width without restructuring. The dashboard column proportion shifted to 1.4fr/0.85fr/1fr to give Standings the space its 7 columns need.

Unchanged

Recent podiums — already 3-line block per season, content earns the column width.
Records — 3-line block per record (eyebrow + title + sub + arrow), already production-pattern-aligned.
Season archive — pure navigation aid, density isn't the goal here.

Data dependencies surfaced by this pass

Several of the line-2 additions require data the existing selectors may not produce. Worth confirming each:

  • Championships line-2 "career win%": already in selectAllTimeStandings
  • Championships line-2 "all-time wins leader" tag: derivable; one-time computation per page render
  • Top rivalries line-2 "last met YEAR (winner)": requires sorting matchup history by date and taking the latest. Plausibly already in the rivalry selector data; if not, small extension
  • Title droughts line-2 "X top-3 finishes · best #N (year)": derivable from per-season final standings. Likely already accessible via existing selectors but may need a thin reshape helper
  • Movers line-2 "won title" annotation: trivial — just check if the destination rank is #1 and the season had a champion

If any of these prove harder to wire than expected, the multi-line layout still works without the line-2 enrichment — drop the editorial copy and just surface what's available. The structural improvement (rows reading as paragraphs instead of drifty single lines) holds regardless.

Honest tradeoffs

Path C is more work than Path A (constrain width). It's also more work than the previous mockup we settled on. The benefit is that the page actually uses its width rather than asking the user to forgive empty space. The cost is real selector work for some of the line-2 additions, and the page becomes taller (each 2-line row is ~36px vs the prior ~30px single-line row).

If the line-2 enrichments prove too costly to implement now, fall back to multi-line stacking with just existing data. Even bare stacking ("name on line 1, count on line 2") solves the drift problem because rows become paragraph-shaped. The enrichment is the cherry on top — the structural fix is the cake.
