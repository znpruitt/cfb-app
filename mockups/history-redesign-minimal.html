What changed in this pass

Removed

  • Top and bottom borders around the Championships section (was a card-style frame)
  • All row-level borders inside Standings, Streaks, Champions, Records, Rivalries, Movers
  • Border-radius pills around Archive items (replaced with text-only links)
  • Section-internal padding that was implying card chrome

Kept

  • Subnav underline (orientation marker — same as production)
  • Column-header underline in data-table (functional — separates labels from data)
  • Whitespace as the only section separator (--space-section: 64px)

Spacing scale

  • --space-section: 64px — between major sections
  • --space-block: 24px — between blocks within a section (e.g., section-head to content)
  • --space-row: 14px — table row vertical padding
  • --space-tight: 8px — tight gaps (eyebrows to titles, header to content)

Why this works

The page now has exactly two horizontal lines visible — the subnav underline and column-header underlines inside tables. Everything else is whitespace + alignment + color. That's the minimum line-count that still preserves table readability. The page reads as one continuous dashboard rather than a stack of bordered sections.

Tradeoff worth flagging

Without row borders, dense tables (like the 8-row Standings) lean heavily on alignment and tabular numbers to stay readable. If users find them hard to scan, the right answer is zebra-striping (alternating row backgrounds at very low contrast, ~3% white) — not bringing back row borders. Default to no striping; add it only if standings becomes a complaint.
